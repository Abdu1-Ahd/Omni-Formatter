# Comprehensive Formatting Rules for Markdown

This document outlines the strict formatting, parsing, and tokenization rules for Markdown. It is designed to act as a reference for syntax highlighters, parsers, and AST (Abstract Syntax Tree) generation tools.

## 1. Core Parsing Logic
The core parser for Markdown operates under the `docs` paradigm.
Block-level parsing (paragraphs, headers, lists) followed by inline parsing (bold, links).

**Tokenization Strategy:**
- The lexer identifies keywords, operators, and literals specific to Markdown.
- Statement boundaries are resolved via: `newline`.
- For tree construction, the parser strictly evaluates scope depth and applies semantic checks pertinent to the `docs` ecosystem.

## 2. Block Element Rules
Block elements encapsulate logical groupings of statements, such as function bodies, loops, conditionals, or data structures.

**Scope Delimiters:**
- **Start Token:** `newline`
- **End Token:** `newline`

**Block-Level Comments:**
Block comments are designed to span multiple lines without terminating the parser context abruptly.
- **Syntax:** `<!-- ... -->`

**Block Rule Example:**
```
# Header
<!-- comment -->
**bold** text.
```

## 3. Inline Element Rules
Inline elements operate within a single line or statement. They encompass arithmetic operations, variable assignments, function invocations, and single-line annotations.

**Inline Comments:**
Inline comments terminate at the end of the line (newline character).
- **Syntax:** `<!--`

**String Literals:**
String boundaries must be strictly respected to prevent injection or parser failure.
- **Supported Enclosures:** `N/A`

## 4. Escape Characters
Within string literals or regex definitions, escape characters are strictly evaluated to emit special, non-printable characters or to escape literal delimiters.

**Supported Escapes:**
- `\\*, \\_, \\[, \\], \\(`

When an escape character is encountered, the lexer consumes the subsequent character as a literal or control code.

## 5. Comprehensive Syntax Example
Below is a clear syntax example demonstrating the interplay of core parsing logic, block elements, inline elements, and string escapes:

```
# Header
<!-- comment -->
**bold** text.
```
